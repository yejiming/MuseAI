## 简介

MuseAI 是一款运行在你本地电脑上的 AI 写作桌面应用，无需上传文件到云端，你的故事始终在你手中。

它不是简单的聊天机器人，而是深度理解你写作工作流的**创作搭档**：

- 帮你构思和修改大纲
- 检测并消除文章中的"AI 味"
- 扮演你笔下的角色，与你深度互动
- 陪你玩转文字冒险，推进故事剧情

## 快速开始

### 1. 下载与安装

前往 [Releases 页面](https://github.com/yejiming/MuseAI/releases) 下载最新版本的安装包（`.dmg`/`.exe`），双击安装后即可启动。

**Windows 用户**：安装前请**关闭杀毒软件**（如 360、腾讯电脑管家、火绒、Windows Defender 等），否则安装程序可能被误拦截导致安装失败或运行异常。安装完成后可重新开启。

macOS 用户安装后如果提示"已损坏，无法打开"，请打开终端运行以下命令后再启动：

```bash
xattr -dr com.apple.quarantine /Applications/MuseAI.app
```

### 2. 配置 AI 模型

首次启动后，进入 **设置** 页面，配置你的 AI 模型：

1. 填写 API Key（支持 OpenAI、Anthropic 及兼容 OpenAI 格式的第三方服务）
2. 填写 API 地址（如使用国内中转服务，在此填写自定义地址）
3. 选择模型名称（如 `deepseek-v4-flash`、`claude-sonnet-4-6` 等）
4. 点击"测试连接"，确认配置正确

![alt text](assets/设置页.png)

> 💡 你可以为不同模块（写作、去 AI 味、大纲等）分别配置不同的模型和参数，灵活搭配性能与成本。

### 3. 开始创作

- **写作**：点击左侧「作品」，新建或导入你的文件，右侧呼出 AI 对话框即可开始协作。
- **去 AI 味**：在「去 AI 味」模块选择文件，先「检测」，再「去除」。
- **大纲**：在「大纲」模块上传大纲文件，点击「评估」获取评分，再点击「制作」让 AI 优化。
- **智能伴侣**：在「背景设定」中创建世界书和角色卡，再到「伴侣对话」选择角色，开始聊天。

## 核心功能

### ✍️ 作品管理与写作
在统一的工作区中管理你所有的作品文件，内置 Markdown 编辑器，支持实时预览与格式高亮。每个文件都有**多版本历史**，修改前自动备份，随时可以回溯。

### 📚 范文库
导入你喜欢的参考文章，AI 在辅助写作时会自动读取范文库内容，让风格建议更贴合你的口味。

### 🪄 大纲模块
- **大纲评估**：上传你的大纲，AI 给出评分与详细的优化建议。
- **大纲制作**：根据你的需求或评估结果，AI 帮你从零搭建结构完整的章节大纲。
- **反向大纲**：导入一篇完整小说（或长篇作品），AI 自动反向提炼出结构化大纲。支持**200 万字长篇小说分布式并行分析**，几分钟内即可完成全篇拆解。

![alt text](assets/反向大纲.png)

### 🧹 去 AI 味
担心自己的文章读起来太像机器生成？
- **检测**：量化分析 AI 味浓度，逐段指出问题。
- **去除**：根据检测建议，AI 自动重写，让文字更有人情味、更接地气。

### 🧠 智能提取背景设定
上传小说原文或大纲，AI 自动提取结构化世界书与角色卡：
- **世界书**：一键提炼世界观、历史、地理、规则等核心设定
- **角色卡**：自动识别登场角色，生成包含性格、关系、口头禅等完整人设
- 配合**反向大纲**功能，即使是长篇小说也能快速完成全部分析与设定提取

![alt text](assets/生成角色卡.png)

### 🧡 智能伴侣
创建专属于你故事世界的角色卡与世界书，然后与你笔下的角色**真正对话**：
- 角色会记住你们之间的关键事件
- 关系随着对话自然演变
- 每次归档后，角色记忆自动更新

### 💞 羁绊档案
查看你与每个角色之间**真实积累的关系轨迹**：
- **关系概览**：一目了然的关系类型、相处模式与关系底线
- **羁绊时间线**：关键事件以时间轴方式呈现，见证关系演变
- **会话足迹**：只显示与该角色聊过的天，精准归类
- **冒险足迹**：回顾与该角色共同经历过的文字冒险

### 🗺️ 故事冒险
化身地下城主（DM），开启沉浸式文字冒险：
- 选定世界书与角色卡，输入初始剧情
- 以**语言、行为、剧情**三种方式推动故事发展
- AI 担任叙事者，实时演绎场景与角色反应
- 会话历史自动保存，随时继续上次的冒险

### 📡 局域网访问
桌面应用启动后，自动开启局域网服务。在同一 WiFi 下的手机浏览器中打开显示的网址，即可直接使用**智能伴侣、故事冒险和羁绊**功能，无需额外安装 App。

| 功能　　 | 说明　　　　　　　　　　　　　　　　　　　　　　|
| ----------| -------------------------------------------------|

| 版本历史 | 每次 AI 修改前自动创建新版本，可随时查看和恢复　|
| 范文参考 | 在「范文库」中导入参考文章，AI 写作时会自动引用 |

### 去 AI 味

1. 在左侧选择一篇文章
2. 点击「开始检测」—— AI 会分析文章各段落，给出 0-100 的 AI 浓度评分及具体建议
3. 点击「开始去除」—— AI 根据建议逐段重写，输出更自然的版本
4. 新版本自动保存为独立文件，原文不受影响

---

### 大纲模块

**评估流程：**
1. 在「大纲」模块中打开你的大纲文件
2. 点击「评估大纲」，AI 从结构、节奏、冲突设计等维度打分
3. 查看详细建议

**制作流程：**
1. 描述你的故事核心（类型、主角、世界观等）
2. 点击「制作大纲」，AI 生成完整章节结构
3. 可在建议基础上继续对话修改

---

### 智能伴侣

**背景设定：**
- **世界书**：描述你故事世界的规则、地理、历史背景
- **角色卡**：填写角色的姓名、性格、说话风格、与你的关系等字段

**对话与记忆：**
- 每次对话结束后，点击「归档记忆」，AI 会分析本次会话，提炼关键事件，更新角色记忆
- 角色的成长与你们的关系变化会被真实记录

**羁绊档案：**
- 在「羁绊」页面选择任意角色卡，查看你们之间的完整关系档案
- 会话足迹与冒险足迹自动按角色归类，只展示与该角色相关的内容

---

### 故事冒险

1. 点击左侧「故事冒险」
2. 选择世界书、一个或多个角色卡
3. 输入初始剧情（故事从哪里开始）
4. 开始冒险！使用以下三种方式推进故事：
   - **语言**：你说出的话（对话）
   - **行为**：你的动作或行动
   - **剧情**：直接推进剧情走向
5. AI 担任叙事者，实时描写场景反应

## 数据存储说明

MuseAI 的所有数据均存储在**本地**，不上传任何内容到外部服务器（API 调用除外）。

所有用户数据统一存放在 `~/Documents/MuseAI/` 目录下：

| 数据类型　　　　| 存储位置　　　　　　　　　　　　　　　　　　　　　　　　　 |
| -----------------| ------------------------------------------------------------|
| 作品文件　　　　| `~/Documents/MuseAI/articles/`　　　　　　　　　　　　　　 |
| 范文库　　　　　| `~/Documents/MuseAI/references/`　　　　　　　　　　　　　 |
| 大纲文件　　　　| `~/Documents/MuseAI/outline/`　　　　　　　　　　　　　　　|
| 版本历史　　　　| 各文件同目录下的 `.versions/` 文件夹　　　　　　　　　　　 |
| 角色卡 / 世界书 | `~/Documents/MuseAI/config/partner-store.json`　　　　　　 |
| 智能伴侣会话　　| `~/Documents/MuseAI/agent-sessions/partner-session-*.json` |
| 故事冒险会话　　| `~/Documents/MuseAI/agent-sessions/story-session-*.json`　 |
| 写作 Agent 会话 | `~/Documents/MuseAI/agent-sessions/session-*.json`　　　　 |
| 应用设置　　　　| `~/Documents/MuseAI/config/settings-store.json`　　　　　　|
| 各模块 UI 状态　| `~/Documents/MuseAI/config/*.json`　　　　　　　　　　　　 |

> ⚠️ 建议定期备份 `~/Documents/MuseAI/` 目录，以防数据丢失。

## 常见问题

**Q：支持哪些 AI 服务商？**

A：支持 OpenAI、Anthropic（Claude），以及所有兼容 OpenAI 接口格式的第三方服务（如 DeepSeek、Kimi、智谱、通义千问等国内中转服务）。只要在设置页填入对应的 API 地址、Key 和模型名即可使用。

**Q：API Key 是什么？我要怎么获取？**

A：API Key 是 AI 服务商提供的一把"钥匙"，用来验证你有权限调用他们的模型。你可以在你选择的 AI 服务商官网注册账号后，在「开发者中心」或「API 管理」页面找到并复制它。 MuseAI 本身不会向你收费，费用由服务商根据你的实际使用量收取。

**Q：为什么连接测试失败？**

A：常见原因有以下几种：
- **API Key 填错了**：请检查是否有多余的空格或漏复制了字符。
- **API 地址填错了**：如果你使用的是国内中转服务，请确认地址完整（通常以 `https://` 开头，以 `/v1` 结尾）。
- **网络问题**：部分服务商需要科学上网才能访问，请确认你的网络环境正常。
- **余额不足**：有些服务商需要在账户里充值后才能调用 API。

**Q：反向大纲提取失败是什么原因？**

A：最常见的原因是 **AI 输出的 Token 不够，导致返回内容被截断**。反向大纲需要把整篇小说浓缩成结构化大纲，即使系统提示词中要求不要超过 10000 字，模型也不一定完全遵守，因此有一定失败概率。

**解决方法**：
- 进入「设置」→「大纲页设置」，找到反向大纲相关的 Agent（短篇 / 长篇-分段摘要 / 长篇-汇总大纲），把「最大输出 Token」尽量调大。
- 同时可以修改该系统提示词，把字数限制和要求说得更严厉、更具体一些，降低模型"超纲"的概率。

**Q：世界书、角色卡提取失败是什么原因？**

A：通常有两种原因：
- **模型服务不稳定**：偶尔会出现网络抖动或服务商响应异常，导致没有返回完整结果。
- **模型没有返回正确格式**：提取世界书和角色卡需要模型输出严格的 JSON 格式，如果模型"不听话"，就可能解析失败。

**解决方法**：直接点击重试即可。如果是提取角色卡，你可以在输入框里**手动列出角色姓名列表**，让 AI 按名单逐个提取，成功率会显著提高。

**Q：封存记忆提取失败是什么原因？**

A：和世界书/角色卡提取类似，封存记忆也需要模型输出严格的 JSON 格式，偶尔会因为服务商不稳定或模型格式偏差导致失败。

**解决方法**：先尝试重试。如果多次失败，建议进入「设置」→「羁绊页设置」，把「聊天页-封存记忆」或「冒险页-封存记忆」的 **Temperature（温度）改成 0**，这样模型输出会更稳定、更听话。

**Q：各个 Agent 的参数应该怎么设置？**

A：大部分情况下，**使用默认参数即可**，不需要手动调整。如果你在某次调整后发现效果变差了，或者记不清自己改了什么，直接点击该 Agent 设置卡片下方的「恢复默认」按钮，一键回到出厂配置。

**Q：我的文章内容会被上传到云端吗？**

A：不会。你的作品文件、范文库、大纲、角色卡等所有数据都保存在你自己的电脑上（`~/Documents/MuseAI/` 文件夹内）。只有你主动点击"发送"让 AI 帮你分析或修改时，相关内容才会通过 API 发送给你配置的 AI 服务商，本地文件不会自动上传。

**Q：去 AI 味检测后，原文会被修改吗？**

A：不会。检测只是分析并给出评分和建议，原文不受影响。如果你点击了"去除 AI 味"，AI 会生成一个新的修改版本，并自动保存为一个独立的新文件，原文始终保留。

**Q：版本历史如何恢复？**

A：在左侧文件列表中找到目标文件，右键点击选择「版本历史」，即可查看该文件的所有历史版本，并选择任意一个版本进行恢复。

**Q：世界书和角色卡有什么区别？应该怎么填？**

A：
- **世界书**描述的是整个故事世界的背景设定，比如时代、地理、规则、势力分布等。同一部作品下的多个角色可以共用同一本世界书。
- **角色卡**描述的是具体某一个人物，包括姓名、性格、说话风格、与你的关系、底线、关键事件等。

填写时不需要追求完整，先写上最关键的信息即可，后续可以在对话中通过「归档记忆」让 AI 自动帮你补充和更新。

**Q：聊天时角色突然"出戏"了，回复不像设定中的人物怎么办？**

A：
- 检查一下是否绑定了正确的**世界书**和**角色卡**，没有绑定设定的话 AI 会按通用模式回复。
- 如果对话已经进行了很长，可能是上下文太长导致 AI "遗忘"了早期设定。可以尝试开启一个新会话，或者调整设置页中聊天 Agent 的「最大上下文 Token」。
- 你也可以在设置页中调整聊天 Agent 的「系统提示词」和「温度」，让角色扮演更稳定或更富变化。

**Q：聊天页和冒险页都能绑定角色卡，有什么区别？**

A：
- **聊天页（智能伴侣）**：适合一对一沉浸式对话，一次只能绑定**一个**角色卡。对话结束后可以「封存记忆」，AI 会自动分析整场对话并更新该角色的关系设定和关键事件。
- **冒险页（故事冒险）**：适合跑团式文字冒险，可以绑定**多个**角色卡和一个世界书。AI 会扮演叙事者（DM），根据你的语言和行动推进剧情，多个角色会在故事中互动。

**Q：手机和电脑的数据是同步的吗？**

A：电脑端的数据保存在本地。手机通过**局域网访问**使用时，数据也存储在电脑本地，不是云同步。也就是说，你在手机上聊的内容，关闭后下次再用手机访问，记录依然保留在电脑上。只要电脑不关机、不删除数据，手机和电脑看到的是同一份数据。

**Q：局域网访问怎么用？手机和电脑需要在同一个 WiFi 吗？**

A：是的，手机需要和电脑连接**同一个 WiFi（或同一个路由器）**。桌面应用启动后会自动开启局域网服务，你只需在电脑的「设置」页面找到显示的网址，用手机浏览器打开即可使用智能伴侣、故事冒险和羁绊功能，无需安装任何 App。

**Q：怎么备份我的数据？**

A：最简单的方法是直接复制你电脑上的 `~/Documents/MuseAI/` 整个文件夹，保存到 U 盘、移动硬盘或网盘里。这个文件夹包含了你的所有作品、角色卡、会话记录和设置。换电脑时，把这个文件夹复制到新电脑的相同位置即可完整恢复。

**Q：为什么 AI 回复很慢，或者中途突然停止了？**

A：
- **模型本身慢**：不同服务商和模型的响应速度差异很大，DeepSeek、Kimi 等国内模型通常比 OpenAI 快。
- **网络波动**：如果网络不稳定，可能会出现响应中断，可以尝试重新发送。
- **Token 限制**：如果当前对话太长，接近了设置中的「最大输出 Token」或「最大上下文 Token」上限，AI 可能会提前截断回复。可以在设置页调大对应 Agent 的 Token 上限。
- **服务商繁忙**：高峰期某些服务商可能会排队或限流，稍等片刻再试即可。

**Q：模型费用怎么算？**

A：MuseAI 本身完全免费，AI 功能的费用由你使用的 API 服务商根据实际调用量直接计费。大部分服务商采用"按 Token 计费"的模式（可以理解为按字数收费），具体价格请参考各服务商官网。你可以在设置页中为不同模块配置不同的模型，比如用便宜快速的模型做日常写作，用高质量模型做关键评估。
