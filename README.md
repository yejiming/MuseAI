# MuseAI — 本地 AI 伴侣、文字冒险与穿书互动应用

[English](README_EN.md) | 简体中文

> 创建你的 AI 角色，进入你的故事世界。
> 支持 AI 伴侣聊天、角色卡、世界书、长期记忆、文字冒险、穿书剧情和本地数据保存。

MuseAI 是一款面向沉浸式角色互动的本地 AI 应用。你可以创建自己的角色与世界设定，和角色持续对话、共同经历冒险，并在一次次互动中积累关系、记忆与羁绊。

它不只是普通的 AI 聊天工具，也不只是写作软件。MuseAI 更像一个可以保存世界、角色、记忆和关系的本地 AI 互动世界。

## MuseAI 适合谁

- 想拥有本地 AI 伴侣、虚拟角色或长期角色聊天体验的用户
- 喜欢角色卡、世界书、AI 跑团、文字冒险和沉浸式剧情互动的玩家
- 想进入自己整理的小说世界，体验穿书剧情和分支选择的创作者
- 需要保存角色记忆、关系变化、关键事件和羁绊轨迹的用户
- 希望作品、设定和会话数据保存在本地电脑，只通过自己的 API Key 调用模型的人

## 核心体验

### AI 伴侣与角色聊天

在「背景设定」中创建世界书和角色卡后，就可以到「伴侣对话」选择角色，开启一对一 AI 角色聊天。角色会根据人设、说话风格、与你的关系和过往记忆回应你，让对话更像一段持续发展的关系，而不是一次性的问答。

- 支持自定义角色姓名、性格、关系、底线、口头禅和背景故事
- 支持绑定世界书，让角色始终处在同一个故事世界中
- 支持归档记忆，提炼关键事件并更新角色卡
- 适合 AI 伴侣、AI 虚拟角色、OC 聊天、角色扮演和沉浸式陪伴

![MuseAI 智能伴侣：选择角色卡后与 AI 角色进行长期对话](assets/聊天页.png)

### 长期记忆与羁绊档案

每次对话或冒险结束后，你都可以让 MuseAI 分析本次经历，将关系变化、重要事件和相处模式写回角色档案。「羁绊」页面会按角色整理这些记录，形成可回顾的关系轨迹。

- **关系概览**：查看关系类型、相处模式与关系底线
- **羁绊时间线**：用时间轴记录关键事件
- **会话足迹**：只展示与该角色相关的聊天记录
- **冒险足迹**：回顾与该角色共同经历过的文字冒险

![MuseAI 封存记忆：从聊天和冒险中提炼角色记忆与羁绊变化](assets/封存记忆.png)

![MuseAI 羁绊档案：查看角色关系轨迹、时间线和共同经历](assets/羁绊页.png)

### AI 文字冒险与跑团

在「冒险」中选择世界书和多个角色卡，MuseAI 会作为故事主持人（DM/GM）推进剧情。你可以用语言、行为或剧情指令推动故事，角色会根据各自人设参与互动。

- 支持一个世界书绑定多个角色卡
- 支持语言、行为、剧情三种输入方式
- AI 负责场景描写、角色反应和剧情推进
- 冒险记录可以归档为角色记忆，沉淀为羁绊事件

![MuseAI 文字冒险：绑定世界书和角色卡后开启 AI 跑团剧情](assets/冒险页.png)

### 穿书互动

「穿书」模式可以把大纲、世界书和角色卡装配成可游玩的故事世界。你可以选择入场点和可扮演身份，进入小说世界后通过对话、行动和剧情选择影响故事走向。

- 支持从已整理的大纲、世界书和角色卡生成穿书素材
- 支持入场导演生成穿书入口和可扮演身份
- 支持场景、节拍、状态和摘要记忆，让长线剧情可以继续推进
- 支持保存穿书进度，后续继续当前剧情

穿书模式适合想体验「进入小说世界」「改写剧情」「和角色共同经历关键事件」的用户。

### 世界书与角色卡

MuseAI 支持手动创建世界书和角色卡，也可以上传小说原文或大纲，让 AI 自动提取结构化设定。

- **世界书**：提炼世界观、时代背景、地理格局、势力分布、规则和核心冲突
- **角色卡**：识别角色姓名、性格、人际关系、说话方式、关键事件和与你的关系
- **设定复用**：同一个世界书和角色卡可以用于伴侣聊天、文字冒险、穿书和羁绊档案

![MuseAI 背景设定：管理世界书、角色卡和故事世界资料](assets/背景页.png)

![MuseAI 角色卡生成：从小说原文或大纲中自动提取角色设定](assets/生成角色卡.png)

### 局域网移动端访问

桌面应用启动后，可以在同一 WiFi 下通过手机浏览器访问 MuseAI。你可以在手机上继续使用智能伴侣、故事冒险和羁绊功能，无需额外安装 App。

![MuseAI 局域网访问：手机浏览器连接电脑端继续聊天和冒险](assets/局域网.png)

## 辅助创作功能

虽然 MuseAI 的核心体验是 AI 伴侣、冒险和穿书，它也内置了一组面向小说和内容创作的辅助工具，方便你整理素材、完善设定并准备可互动的故事世界。

### 作品管理与 Markdown 编辑

在统一的工作区中管理作品文件，内置 Markdown 编辑器，支持实时预览与格式高亮。每个文件都有多版本历史，修改前自动备份，随时可以回溯。

![MuseAI 作品管理：本地文件树、Markdown 编辑器和 AI 对话工作区](assets/作品页.png)

### 范文库

导入你喜欢的参考文章，AI 在辅助写作或分析风格时可以读取范文库内容，让建议更贴合你的口味。

### 大纲评估、制作与反向大纲

- **大纲评估**：上传大纲，AI 给出评分与详细优化建议
- **大纲制作**：根据故事需求生成章节结构
- **反向大纲**：导入完整小说或长篇作品，AI 自动反向提炼结构化大纲
- **长篇分析**：支持长篇小说分布式并行分析，便于后续生成世界书、角色卡和穿书素材

![MuseAI 大纲模块：评估大纲、制作大纲和整理故事结构](assets/大纲页.png)

![MuseAI 反向大纲：从长篇小说中提炼结构化大纲](assets/反向大纲.png)

### 去 AI 味

如果你的文章读起来太像机器生成，可以用「去 AI 味」模块先检测，再根据建议生成更自然的修改版本。原文不会被直接覆盖。

- **检测**：量化分析 AI 味浓度，逐段指出问题
- **去除**：根据检测建议自动重写，让文字更自然、更接地气

## 快速开始

### 1. 下载与安装

前往 [Releases 页面](https://github.com/yejiming/MuseAI/releases) 下载最新版本的安装包（`.dmg`/`.exe`），双击安装后即可启动。

**Windows 用户**：安装前请关闭杀毒软件（如 360、腾讯电脑管家、火绒、Windows Defender 等），否则安装程序可能被误拦截导致安装失败或运行异常。安装完成后可重新开启。

macOS 用户安装后如果提示「已损坏，无法打开」，请打开终端运行以下命令后再启动：

```bash
xattr -dr com.apple.quarantine /Applications/MuseAI.app
```

### 2. 配置 AI 模型

首次启动后，进入「设置」页面，配置你的 AI 模型：

1. 填写 API Key（支持 OpenAI、Anthropic 及兼容 OpenAI 格式的第三方服务）
2. 填写 API 地址（如使用国内中转服务，在此填写自定义地址）
3. 选择模型名称（如 `deepseek-v4-flash`、`claude-sonnet-4-6` 等）
4. 点击「测试连接」，确认配置正确

![MuseAI 设置页：配置 OpenAI、Claude、DeepSeek、Kimi 等模型服务](assets/设置页.png)

你可以为伴侣聊天、冒险、穿书、去 AI 味、大纲等模块分别配置不同模型和参数，灵活搭配效果、速度与成本。

### 3. 开始使用

- **AI 伴侣**：在「背景设定」创建世界书和角色卡，再到「伴侣对话」选择角色开始聊天
- **文字冒险**：在「冒险」选择世界书和角色卡，输入开场剧情后开始 AI 跑团
- **穿书互动**：先到「穿书素材」装配大纲、世界书和角色卡，再进入「穿书」开始剧情
- **羁绊档案**：对话或冒险归档后，到「羁绊」查看关系变化和关键事件
- **辅助创作**：在「作品」「大纲」「去 AI 味」中整理文本、评估结构和优化表达

## 数据存储说明

MuseAI 的所有数据均存储在本地，不上传任何内容到外部服务器（API 调用除外）。

所有用户数据统一存放在 `~/Documents/MuseAI/` 目录下：

| 数据类型 | 存储位置 |
| --- | --- |
| 作品文件 | `~/Documents/MuseAI/articles/` |
| 范文库 | `~/Documents/MuseAI/references/` |
| 大纲文件 | `~/Documents/MuseAI/outline/` |
| 版本历史 | 各文件同目录下的 `.versions/` 文件夹 |
| 角色卡 / 世界书 | `~/Documents/MuseAI/config/partner-store.json` |
| 智能伴侣会话 | `~/Documents/MuseAI/agent-sessions/partner-session-*.json` |
| 故事冒险会话 | `~/Documents/MuseAI/agent-sessions/story-session-*.json` |
| 写作 Agent 会话 | `~/Documents/MuseAI/agent-sessions/session-*.json` |
| 应用设置 | `~/Documents/MuseAI/config/settings-store.json` |
| 各模块 UI 状态 | `~/Documents/MuseAI/config/*.json` |

建议定期备份 `~/Documents/MuseAI/` 目录，以防数据丢失。

## 常见问题

**Q：MuseAI 和普通 AI 聊天工具有什么区别？**

A：普通 AI 聊天工具通常以单次对话为主，角色、世界观和记忆容易散落在不同窗口里。MuseAI 会把世界书、角色卡、聊天记录、冒险记录和羁绊变化保存在本地，让同一个角色可以长期延续。

**Q：MuseAI 是 AI 伴侣应用还是 AI 写作工具？**

A：MuseAI 当前更适合作为本地 AI 伴侣、AI 角色聊天、文字冒险和穿书互动应用使用。作品管理、大纲、反向大纲和去 AI 味属于辅助创作能力，用来帮助你整理故事世界和互动素材。

**Q：支持哪些 AI 服务商？**

A：支持 OpenAI、Anthropic（Claude），以及所有兼容 OpenAI 接口格式的第三方服务（如 DeepSeek、Kimi、智谱、通义千问等国内中转服务）。只要在设置页填入对应的 API 地址、Key 和模型名即可使用。

**Q：API Key 是什么？我要怎么获取？**

A：API Key 是 AI 服务商提供的一把「钥匙」，用来验证你有权限调用他们的模型。你可以在你选择的 AI 服务商官网注册账号后，在「开发者中心」或「API 管理」页面找到并复制它。MuseAI 本身不会向你收费，费用由服务商根据你的实际使用量收取。

**Q：为什么连接测试失败？**

A：常见原因有以下几种：

- **API Key 填错了**：请检查是否有多余的空格或漏复制了字符。
- **API 地址填错了**：如果你使用的是国内中转服务，请确认地址完整（通常以 `https://` 开头，以 `/v1` 结尾）。
- **网络问题**：部分服务商需要科学上网才能访问，请确认你的网络环境正常。
- **余额不足**：有些服务商需要在账户里充值后才能调用 API。

**Q：我的文章内容、角色卡和聊天记录会被上传到云端吗？**

A：不会自动上传。你的作品文件、范文库、大纲、角色卡、世界书和会话记录都保存在你自己的电脑上（`~/Documents/MuseAI/` 文件夹内）。只有你主动发送消息、分析文本或生成内容时，相关上下文才会通过 API 发送给你配置的 AI 服务商。

**Q：世界书和角色卡有什么区别？应该怎么填？**

A：

- **世界书**描述的是整个故事世界的背景设定，比如时代、地理、规则、势力分布和核心冲突。同一部作品下的多个角色可以共用同一本世界书。
- **角色卡**描述的是具体某一个人物，包括姓名、性格、说话风格、与你的关系、底线、关键事件等。

填写时不需要追求完整，先写上最关键的信息即可，后续可以在对话中通过「归档记忆」让 AI 自动帮你补充和更新。

**Q：聊天时角色突然「出戏」了，回复不像设定中的人物怎么办？**

A：

- 检查是否绑定了正确的世界书和角色卡，没有绑定设定的话 AI 会按通用模式回复。
- 如果对话已经进行了很长，可能是上下文太长导致 AI 遗忘早期设定。可以尝试开启一个新会话，或者调整设置页中聊天 Agent 的「最大上下文 Token」。
- 你也可以在设置页中调整聊天 Agent 的「系统提示词」和「温度」，让角色扮演更稳定或更富变化。

**Q：聊天、冒险和穿书有什么区别？**

A：

- **聊天页（智能伴侣）**：适合一对一沉浸式对话，一次只能绑定一个角色卡。对话结束后可以归档记忆，更新该角色的关系设定和关键事件。
- **冒险页（文字冒险）**：适合跑团式剧情，可以绑定多个角色卡和一个世界书。AI 会扮演叙事者（DM/GM），根据你的语言、行为和剧情指令推进故事。
- **穿书页（穿书互动）**：适合进入一个已装配好的小说世界。你可以选择入场点和身份，在场景与节拍中参与剧情，并保存进度后继续游玩。

**Q：反向大纲提取失败是什么原因？**

A：最常见的原因是 AI 输出的 Token 不够，导致返回内容被截断。反向大纲需要把整篇小说浓缩成结构化大纲，即使系统提示词中要求不要超过 10000 字，模型也不一定完全遵守，因此有一定失败概率。

**解决方法**：

- 进入「设置」→「大纲页设置」，找到反向大纲相关的 Agent（短篇 / 长篇-分段摘要 / 长篇-汇总大纲），把「最大输出 Token」尽量调大。
- 同时可以修改该系统提示词，把字数限制和要求说得更严厉、更具体一些，降低模型超纲的概率。

**Q：世界书、角色卡提取失败是什么原因？**

A：通常有两种原因：

- **模型服务不稳定**：偶尔会出现网络抖动或服务商响应异常，导致没有返回完整结果。
- **模型没有返回正确格式**：提取世界书和角色卡需要模型输出严格的 JSON 格式，如果模型没有按格式返回，就可能解析失败。

**解决方法**：直接点击重试即可。如果是提取角色卡，你可以在输入框里手动列出角色姓名列表，让 AI 按名单逐个提取，成功率会显著提高。

**Q：封存记忆提取失败是什么原因？**

A：和世界书/角色卡提取类似，封存记忆也需要模型输出严格的 JSON 格式，偶尔会因为服务商不稳定或模型格式偏差导致失败。

**解决方法**：先尝试重试。如果多次失败，建议进入「设置」→「羁绊页设置」，把「聊天页-封存记忆」或「冒险页-封存记忆」的 Temperature（温度）改成 0，这样模型输出会更稳定。

**Q：手机和电脑的数据是同步的吗？**

A：电脑端的数据保存在本地。手机通过局域网访问使用时，数据也存储在电脑本地，不是云同步。也就是说，你在手机上聊的内容，关闭后下次再用手机访问，记录依然保留在电脑上。只要电脑不关机、不删除数据，手机和电脑看到的是同一份数据。

**Q：局域网访问怎么用？手机和电脑需要在同一个 WiFi 吗？**

A：是的，手机需要和电脑连接同一个 WiFi（或同一个路由器）。桌面应用启动后会自动开启局域网服务，你只需在电脑的「设置」页面找到显示的网址，用手机浏览器打开即可使用智能伴侣、故事冒险和羁绊功能，无需安装任何 App。

**Q：去 AI 味检测后，原文会被修改吗？**

A：不会。检测只是分析并给出评分和建议，原文不受影响。如果你点击「去除 AI 味」，AI 会生成一个新的修改版本，并自动保存为一个独立的新文件，原文始终保留。

**Q：版本历史如何恢复？**

A：在左侧文件列表中找到目标文件，右键点击选择「版本历史」，即可查看该文件的所有历史版本，并选择任意一个版本进行恢复。

**Q：怎么备份我的数据？**

A：最简单的方法是直接复制你电脑上的 `~/Documents/MuseAI/` 整个文件夹，保存到 U 盘、移动硬盘或网盘里。这个文件夹包含了你的所有作品、角色卡、会话记录和设置。换电脑时，把这个文件夹复制到新电脑的相同位置即可完整恢复。

**Q：为什么 AI 回复很慢，或者中途突然停止了？**

A：

- **模型本身慢**：不同服务商和模型的响应速度差异很大，DeepSeek、Kimi 等国内模型通常比 OpenAI 快。
- **网络波动**：如果网络不稳定，可能会出现响应中断，可以尝试重新发送。
- **Token 限制**：如果当前对话太长，接近了设置中的「最大输出 Token」或「最大上下文 Token」上限，AI 可能会提前截断回复。可以在设置页调大对应 Agent 的 Token 上限。
- **服务商繁忙**：高峰期某些服务商可能会排队或限流，稍等片刻再试即可。

**Q：模型费用怎么算？**

A：MuseAI 本身完全免费，AI 功能的费用由你使用的 API 服务商根据实际调用量直接计费。大部分服务商采用按 Token 计费的模式（可以理解为按字数收费），具体价格请参考各服务商官网。你可以在设置页中为不同模块配置不同的模型，比如用便宜快速的模型做日常聊天，用高质量模型做关键剧情和设定分析。
