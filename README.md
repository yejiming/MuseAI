## 简介

MuseAI 是一款运行在你本地电脑上的 AI 写作桌面应用，无需上传文件到云端，你的故事始终在你手中。

它不是简单的聊天机器人，而是深度理解你写作工作流的**创作搭档**：

- 帮你构思和修改大纲
- 检测并消除文章中的"AI 味"
- 扮演你笔下的角色，与你深度互动
- 陪你玩转文字冒险，推进故事剧情

## 核心功能

### ✍️ 作品管理与写作
在统一的工作区中管理你所有的作品文件，内置 Markdown 编辑器，支持实时预览与格式高亮。每个文件都有**多版本历史**，修改前自动备份，随时可以回溯。

### 📚 范文库
导入你喜欢的参考文章，AI 在辅助写作时会自动读取范文库内容，让风格建议更贴合你的口味。

### 🪄 大纲模块
- **大纲评估**：上传你的大纲，AI 给出评分与详细的优化建议。
- **大纲制作**：根据你的需求或评估结果，AI 帮你从零搭建结构完整的章节大纲。

### 🧹 去 AI 味
担心自己的文章读起来太像机器生成？
- **检测**：量化分析 AI 味浓度，逐段指出问题。
- **去除**：根据检测建议，AI 自动重写，让文字更有人情味、更接地气。

### 🧡 智能伴侣
创建专属于你故事世界的角色卡与世界书，然后与你笔下的角色**真正对话**：
- 角色会记住你们之间的关键事件
- 关系随着对话自然演变
- 每次归档后，角色记忆自动更新

### 💞 羁绊档案
查看你与每个角色之间**真实积累的关系轨迹**：
- **关系概览**：一目了然的关系类型、相处模式与关系底线
- **羁绊时间线**：关键事件以时间轴方式呈现，见证关系演变
- **会话足迹**：只显示与该角色聊过的天，精准归类
- **冒险足迹**：回顾与该角色共同经历过的文字冒险

### 🗺️ 故事冒险
化身地下城主（DM），开启沉浸式文字冒险：
- 选定世界书与角色卡，输入初始剧情
- 以**语言、行为、剧情**三种方式推动故事发展
- AI 担任叙事者，实时演绎场景与角色反应
- 会话历史自动保存，随时继续上次的冒险

## 快速开始

### 1. 下载与安装

前往 [Releases 页面](https://github.com/yejiming/MuseAI/releases) 下载最新版本的安装包（`.dmg`/`.exe`），双击安装后即可启动。

macOS 用户安装后如果提示"已损坏，无法打开"，请打开终端运行以下命令后再启动：

```bash
xattr -dr com.apple.quarantine /Applications/MuseAI.app
```

### 2. 配置 AI 模型

首次启动后，进入 **设置** 页面，配置你的 AI 模型：

1. 填写 API Key（支持 OpenAI、Anthropic 及兼容 OpenAI 格式的第三方服务）
2. 填写 API 地址（如使用国内中转服务，在此填写自定义地址）
3. 选择模型名称（如 `deepseek-v4-flash`、`claude-sonnet-4-6` 等）
4. 点击"测试连接"，确认配置正确

![alt text](assets/设置页.png)

> 💡 你可以为不同模块（写作、去 AI 味、大纲等）分别配置不同的模型和参数，灵活搭配性能与成本。

### 3. 开始创作

- **写作**：点击左侧「作品」，新建或导入你的文件，右侧呼出 AI 对话框即可开始协作。
- **去 AI 味**：在「去 AI 味」模块选择文件，先「检测」，再「去除」。
- **大纲**：在「大纲」模块上传大纲文件，点击「评估」获取评分，再点击「制作」让 AI 优化。
- **智能伴侣**：在「背景设定」中创建世界书和角色卡，再到「伴侣对话」选择角色，开始聊天。

## 各模块详解

### 作品管理

所有作品文件保存在 `~/Documents/MuseAI/articles/` 目录下，你也可以直接用 Finder 管理这个文件夹。

| 功能　　 | 说明　　　　　　　　　　　　　　　　　　　　　　|
| ----------| -------------------------------------------------|
| 新建文件 | 在左侧文件树中右键，选择「新建文件」　　　　　　|
| 版本历史 | 每次 AI 修改前自动创建新版本，可随时查看和恢复　|
| 范文参考 | 在「范文库」中导入参考文章，AI 写作时会自动引用 |

---

### 去 AI 味

1. 在左侧选择一篇文章
2. 点击「开始检测」—— AI 会分析文章各段落，给出 0-100 的 AI 浓度评分及具体建议
3. 点击「开始去除」—— AI 根据建议逐段重写，输出更自然的版本
4. 新版本自动保存为独立文件，原文不受影响

---

### 大纲模块

**评估流程：**
1. 在「大纲」模块中打开你的大纲文件
2. 点击「评估大纲」，AI 从结构、节奏、冲突设计等维度打分
3. 查看详细建议

**制作流程：**
1. 描述你的故事核心（类型、主角、世界观等）
2. 点击「制作大纲」，AI 生成完整章节结构
3. 可在建议基础上继续对话修改

---

### 智能伴侣

**背景设定：**
- **世界书**：描述你故事世界的规则、地理、历史背景
- **角色卡**：填写角色的姓名、性格、说话风格、与你的关系等字段

**对话与记忆：**
- 每次对话结束后，点击「归档记忆」，AI 会分析本次会话，提炼关键事件，更新角色记忆
- 角色的成长与你们的关系变化会被真实记录

**羁绊档案：**
- 在「羁绊」页面选择任意角色卡，查看你们之间的完整关系档案
- 会话足迹与冒险足迹自动按角色归类，只展示与该角色相关的内容

---

### 故事冒险

1. 点击左侧「故事冒险」
2. 选择世界书、一个或多个角色卡
3. 输入初始剧情（故事从哪里开始）
4. 开始冒险！使用以下三种方式推进故事：
   - **语言**：你说出的话（对话）
   - **行为**：你的动作或行动
   - **剧情**：直接推进剧情走向
5. AI 担任叙事者，实时描写场景反应

## 数据存储说明

MuseAI 的所有数据均存储在**本地**，不上传任何内容到外部服务器（API 调用除外）。

所有用户数据统一存放在 `~/Documents/MuseAI/` 目录下：

| 数据类型 | 存储位置 |
|----------|----------|
| 作品文件 | `~/Documents/MuseAI/articles/` |
| 范文库 | `~/Documents/MuseAI/references/` |
| 大纲文件 | `~/Documents/MuseAI/outline/` |
| 版本历史 | 各文件同目录下的 `.versions/` 文件夹 |
| 角色卡 / 世界书 | `~/Documents/MuseAI/config/partner-store.json` |
| 智能伴侣会话 | `~/Documents/MuseAI/agent-sessions/partner-session-*.json` |
| 故事冒险会话 | `~/Documents/MuseAI/agent-sessions/story-session-*.json` |
| 写作 Agent 会话 | `~/Documents/MuseAI/agent-sessions/session-*.json` |
| 应用设置 | `~/Documents/MuseAI/config/settings-store.json` |
| 各模块 UI 状态 | `~/Documents/MuseAI/config/*.json` |

> ⚠️ 建议定期备份 `~/Documents/MuseAI/` 目录，以防数据丢失。

## 常见问题

**Q：支持哪些 AI 服务商？**

A：支持 OpenAI、Anthropic（Claude），以及所有兼容 OpenAI 接口格式的第三方服务（如 DeepSeek、各类国内 API 中转等）。

**Q：我的文章内容会被上传到云端吗？**

A：只有你主动发送给 AI 的内容会通过 API 发送到对应的 AI 服务商，本地文件不会自动上传。

**Q：版本历史如何恢复？**

A：在左侧文件列表中选择文件，点击版本历史图标，即可查看所有历史版本并选择恢复。

**Q：角色卡的记忆数据存储在哪里？**

A：角色卡与世界书保存在 `~/Documents/MuseAI/config/partner-store.json`，会话历史保存在 `~/Documents/MuseAI/agent-sessions/` 下。所有数据均为本地文件，可直接备份整个 `~/Documents/MuseAI/` 目录进行迁移。

**Q：模型费用怎么算？**

A：MuseAI 本身免费，AI 功能的费用由你使用的 API 服务商直接计费，具体价格请参考各服务商官网。
