# 移动端访问令牌输入设计

## 目标

修正移动端首页无论鉴权状态都显示“连接状态：已连接”的误导提示，并让没有 URL token 或本地 token 的用户可以直接在页面中输入访问令牌。

## 交互设计

- 页面加载时读取 URL 或 `localStorage` 中已有的 token。
- 有 token 时，自动调用受保护的会话列表接口验证。
- 没有 token 时，首页显示访问令牌输入框和“验证并连接”按钮。
- 验证中显示“正在验证连接…”并禁用重复提交。
- 验证成功后：
  - 将 token 保存到 `localStorage`。
  - 显示“连接状态：已验证”。
  - 开放伴侣聊天、故事冒险和智能羁绊入口。
- 验证失败后：
  - 不保存输入的 token，并清除无效的本地 token。
  - 显示“访问令牌无效，请检查后重试”。
  - 保持功能入口禁用。

## 技术设计

- 在 `src/utils/runtime.ts` 增加设置和清除移动端 token 的小型工具函数，沿用现有 `mobile_token` 存储键。
- 在 `MobileHome.tsx` 内维护未验证、验证中、已验证和验证失败四种状态。
- 验证请求复用现有 `appInvoke('list_agent_sessions')` 调用链，避免新增后端接口。
- 功能入口在未验证时使用原生 `disabled` 属性，避免仅靠样式阻止导航。
- 所有新增用户可见文案使用中文，视觉沿用现有暖色极简风格。

## 错误处理

- HTTP 401 或其他验证请求失败统一视为未验证，页面显示可操作的中文提示。
- 空白 token 不发起请求，提示用户输入访问令牌。
- 已保存 token 失效时自动清除，避免页面刷新后重复使用错误凭据。

## 测试

- 无 token 时显示输入区，状态不再显示“已连接”，功能入口禁用。
- 输入正确 token 后保存 token、显示“已验证”并开放入口。
- 输入错误 token 后显示错误、清除无效 token并保持入口禁用。
- URL 或本地已有 token 时自动验证。
- 保留现有移动端按钮键盘可访问性和导航测试。
